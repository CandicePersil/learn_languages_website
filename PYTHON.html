Introduction

In this page we will try to show the most interesting facts about Python.

We will focus on the aspect that python can be used to execut specific scripts.

Les fonctions

Elles sont de la forme: fonction(param_1,param_2). Pour les définir, on ajoute le mot clé def en début de fonction.
On utilise la fonction type pour connaître le type d'une variable ou d'un nombre: type(variables).
Cela retourne < class 'type' >
Une autre fonction: print("a=",a,"b=",b) avec a et b des variables ou encore print("Hello World!")

Un autre façon d'écrire les fonctions: utilisation de lambda

  • f = lambda x: x*x
  • f(5) >>>25

  • f1 = lambda x, y: x+y
  • f1(5,4) >>>9

Les structures conditionnelles

  • if a>0 and b=0:
    • print("a>0 et b=0")
  • elif a>0 or b=0:
    • print("on ne sait pas quelle condition est vraie")
  • else:
    • print("a<=0 et b!=0")

Interaction avec l'utilisateur

L'utilisateur écrit quelque chose ensuite et appuie sur entrée. Cela sera stocké dans variable.

Les modules

C'est un bout de code dans un fichier. Il faut importer les modules pour jouir de leur contenu.
Pour importer un module: import module, module.fonction(valeur) #pour utiliser une fontion du module après l'avoir importé.
Par exemple : le module math
Le mot clé help peut être utilisé pour avoir des informations sur un module ou sa fonction.
Il suffit de faire : help("module"), help("module.fonction").
Il existe une autre façon d'importer des modules : from module import fonction # cela permet d'importer seulement cette fonction du module.
Maintenant, la fonction peut être appelée sans faire référence au module: fonction(valeur).

Il faut utiliser un fichier pour y mettre son code python. En effet, écrire son code dans le terminal ne le sauvegarde pas et devient vite rébarbatif.
fichier_python.py
Dans ce fichier la première ligne ce doit d'être: #!chemin vers l'interpréteur de commandes
exemple: #!/usr/bin/python3.4
Une autre ligne peut être placée avant pour encoder les accents:
exemple: # -*- coding: utf-8 -*
Pour finir, il faut éxécuter le fichier python dans le terminal linux.

Pour commenter les modules et fonction, on utlise: les docsting """commentaire"""

Agencement possible des modules:

  • module 1 -> contient les fonctions intéressantes

  • module 2 -> test module 1

Le test peut être fait dans le module lui même.
if _name_ == "_main_":
. fonction(valeur)
Le test est exécuté si on double-clic sur le fichier, au contraire si le fichier est importé le test ne s'exécute pas

Les packages = répertoires

C'est un moyen de regrouper les modules. Ils peuvent être agencé de la sorte:

  • package principal
    • package secondaire 1
      • module 1
      • module 1_1
    • package secondaire 2
      • module 2
      • module 2_1

Pour les importer: import mon_package
import mon_package.sous_package
import package.sous_package_module

Un fichier d'intialisation peut être placé dans le package: __init__.py

Les exceptions

try et except : instruction à tester -> try:
instructions à exécuter en code d'erreur -> except

Forme minimale
  • try:
    • #code à tester
  • except:
    • #print("exception 1")


Il n'y a pas de différentiation entre les différentes exceptions possibles.
Cela ne signifie pas forcément qu'il y a des erreurs.

Forme complète

Gère les différents types d'erreurs.

  • try:
    • #code
  • except CodeErreur1:
    • print("C'est l'erreur 1.")
  • execept CodeErreur2:
    • print("C'est l'erreur 2.")

L'erreur peut être capturée:

  • try:
    • #code
  • except type_d_exception as exception_retournee:
    • print("L'erreur est : ", exception_retournee)

Utilisation de else:

  • try:
    • #code
  • except Exception:
    • print("...")
  • else:
    • print("Le resultat est : ", resultat)

Le mot clé finally:
Après try et except. Cela permet d'exécuter l'intruction de try avec ou sans présence d'erreurs.

Le mot clé pass:
Après except. Rien ne se passe en cas d'erreur.

Les assertions

Cela permet de s'assurer que les exceptions sont respectées.
assert test -> renvoie true -> exception normale
sinon -> assertion error est levée

Pour lever une exception : raise exception("message de l'exception")
